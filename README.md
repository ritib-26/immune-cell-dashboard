# Immune Cell Dashboard
### Miraclib Clinical Trial | Immune Cell Population Analysis

An interactive dashboard and analytical pipeline for exploring immune cell population dynamics in melanoma patients receiving miraclib treatment.

**Live Dashboard:** https://immune-cell-dashboard-6qp9ntvuvfkt59cz4hsgx3.streamlit.app/

---

## How to Run

### Requirements
- Python 3.12+
- Git

### Setup
Clone the repository and install dependencies:
```bash
git clone https://github.com/ritib-26/immune-cell-dashboard.git
cd immune-cell-dashboard
python -m venv env
source env/bin/activate  # Windows: env\Scripts\activate
make setup
```

### Run the pipeline
Initializes the database and generates all outputs:
```bash
make pipeline
```

### Launch the dashboard
```bash
make dashboard
```
Then open http://localhost:8501 in your browser.

## Database Schema

The data is stored in a SQLite database (`cell_counts.db`) with 4 tables:

### `projects`
Stores project-level metadata. One row per project.
| Column | Type | Description |
|--------|------|-------------|
| project_id | TEXT | Primary key (e.g. prj1, prj2) |

### `subjects`
Stores subject-level metadata. One row per unique patient.
| Column | Type | Description |
|--------|------|-------------|
| subject_id | TEXT | Primary key |
| age | INTEGER | Patient age |
| sex | TEXT | Patient sex (M/F) |
| response | TEXT | Treatment response (yes/no) |

### `samples`
Stores one row per biological sample. Links to both projects and subjects.
| Column | Type | Description |
|--------|------|-------------|
| sample_id | TEXT | Primary key |
| subject_id | TEXT | Foreign key → subjects |
| project_id | TEXT | Foreign key → projects |
| condition | TEXT | Disease indication (e.g. melanoma) |
| treatment | TEXT | Drug administered |
| sample_type | TEXT | Sample type (PBMC, WB) |
| time_from_treatment_start | REAL | Timepoint in days |

### `cell_counts`
Stores one row per sample × cell population (narrow/EAV format).
| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER | Auto-incremented primary key |
| sample_id | TEXT | Foreign key → samples |
| population | TEXT | Cell type (b_cell, cd8_t_cell, etc.) |
| count | INTEGER | Raw cell count |

### Schema Rationale
The schema separates concerns across 4 normalized tables to avoid data redundancy. Subject metadata (age, sex, response) is stored once per subject rather than repeated on every sample row. Cell counts use a narrow format (one row per population) rather than wide format (one column per population) — this means adding new cell populations never requires a schema migration.

**Scalability:** This schema scales well to hundreds of projects and thousands of samples because:
- Indexes on `sample_id`, `population`, and `treatment` make filtering fast even at large scale
- The narrow cell_counts table avoids sparse columns if different projects measure different populations
- Projects and subjects are decoupled from samples, so cross-project queries are straightforward
- Adding new metadata fields only requires altering one table

### Design Decisions
- **Separation of concerns:** `load_data.py` only handles the database, `src/analysis.py` only handles queries and statistics, `dashboard.py` only handles the UI. This makes each component independently testable and easy to extend.
- **SQL for filtering, pandas for aggregation:** Complex joins and filters are done in SQL where they're most efficient. Aggregations and statistical tests are done in pandas/scipy where the ecosystem is richer.
- **Streamlit for the dashboard:** Chosen for its simplicity and Python-native workflow. The entire dashboard is one readable script with no frontend code required.
- **Narrow cell_counts table:** Storing one row per sample × population rather than one column per population makes the schema forward-compatible with new cell types and easier to query with GROUP BY.

## Dashboard

The dashboard has three pages:

- **Analysis** — key findings summary, baseline cohort overview, and interactive boxplots comparing responder vs non-responder cell population frequencies
- **Additional Analysis** — time course line plots showing how cell population frequencies evolve across day 0, 7, and 14 of treatment
- **Data Explorer** — filterable table of the full frequency dataset
